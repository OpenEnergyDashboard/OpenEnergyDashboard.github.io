OED is an open project where information about all aspects of the project are maintained on our GitHub site. We welcome, encourage and desire receiving the thoughts, ideas, suggestions, problems or anything else about the OED project. This page gives information on engaging with the OED project.

Discord

The developers of the OED project are available via Discord. You can join with this Discord invite link. Developers are especially encouraged to join. If enough users or traffic happens then we will create multiple channels.

Email contact with the OED project

The OED project maintains a number of email addresses for your input. If you are unsure where to email your input then pick whatever one seems best. It is more important to us that you reach out than worry about how you do it.

  • bug@OpenEnergyDashboard.org: to report a bug or issue. Also see below about submitting issues to GitHub which also has a listing of all known issues.
  • security@OpenEnergyDashboard.org: to report a security issue. More information about reporting OED security issues is available.
  • suggestion@OpenEnergyDashboard.org: to make a suggestion for a new feature, improvement or anything else.
  • help@OpenEnergyDashboard.org: to ask about usage or help in installing OED.
  • info@OpenEnergyDashboard.org: for general inquires and ones not covered by other categories.
  • engage@OpenEnergyDashboard.org: to become involved in the OED project including reviewing features and translation.
  • developer@OpenEnergyDashboard.org: to ask developer questions or to let us know you are a new developer. For information, please see our developer pages.
  • internship@OpenEnergyDashboard.org: to inquire about having an internship with OED. For information, please see our internship page.
  • leadership@OpenEnergyDashboard.org: to reach the people running OED. In addition to any area you want to reach out to the leadership, please use this for concerns about any content in the OED project or if you have concerns about interactions with any member of the OED project.

GitHub

OED resides on GitHub, a platform to store all project information and a history of the code. There are several areas that might be of interest when trying to contact us.

  • The OED GitHub site has an issue area to view all known issues, to submit a new issue or comment on an existing issue. You can search all the issues to see if your concern is already documented and look at the conversation to date. An "Open" issue means it is not yet resolved and a "Closed" issue means it is resolved. Note that a closed issue may not yet be in the version of OED you are running. You can comment on a closed issue or open a new one if you feel the concern has not been adequately or correctly addressed.
  • The OED GitHub site has a discussion area. One is the "User Discussion Area" that allows users to make suggestions and have discussions about OED. This contains a thread about features that have been proposed, their status and allows your input on these features or others you would want. If you have a GitHub account then you can subscribe to an OED Discussion by clicking the button on the right side of any discussion page.

Hearing back from OED

You should expect to hear back from us about your input and how we plan to deal with it. For email we generally contact you directly via email. For input on the GitHub site we generally respond by posting on GitHub.
